# Pattern: definition

**Signal:** the core query is "What is X?" or a topic is a concept/category that needs a clean reference entry.

**Typical length:** 900-1,400 words.

**Citation target:** ≥2 external citations — definitions and scale claims benefit from authoritative backing.

## Section order

1. **Title** — `What is [X]?` or a noun phrase that mirrors the query exactly.

2. **Answer-first block** (no H2, sits directly under H1):
   - Sentence 1: literal definition of X containing the exact target-keyword phrase. 15-25 words.
   - Sentence 2: mechanism or scope. 15-25 words.
   - Combined = 30-50 words. This is what LLMs extract.

3. **TL;DR paragraph** immediately after the answer-first block. 80-120 words. Structure:
   - Definition expanded (1-2 sentences)
   - Mechanism or how it works (1-2 sentences)
   - Outcome or why it matters (1 sentence)
   - Optional proof point (1 sentence — approved stat or external citation)

4. `## How [X] works` — the mechanism in plain language. 150-200 words. One chunk or a short numbered list. If it's a pipeline, describe the stages; if it's a concept, describe the relationships.

5. `## Core components` — a 4-6 row table. Columns: Component, Function, Why it matters. Keep cells concise.

6. `## Why [X] matters` — the business case with at least one approved metric range and at least one external citation. Bullet list or short comparison table.

7. `## Use cases` or `## When to consider [X]` — 3-5 concrete scenarios, one sentence each. Name specific app categories (fitness, learning, finance, creator, e-commerce) where relevant.

8. `## [Brand-driven pitch heading]` — the pitch section. Heading and body are generated from the fetched brand-messaging files (`positioning.md`, `value-story.md`, `boilerplates.md`). The skill does not prescribe a template here. Use the approved customer list (Noom, Harley-Davidson, Smart Fit, Ulta Beauty, Betgames) and approved stats only if they fit the topic naturally. 150-250 words.

9. `## FAQs` — 4-6 question-answer pairs. Questions come from real user research (PAA + Reddit), not invented patterns. Each answer starts directly ("Yes.", "No.", or the definition). Answers are ≤3 sentences. Do not embed source URLs in the document — the output is a Word document. List source URLs in the final chat summary when the skill delivers the batch.

10. `## Conclusion` — 2-3 sentences. Restate the definition in a single sentence, name the outcome, end without a call-to-action.

## Writing rules specific to this pattern

- First sentence is a literal definition: "[X] is [genus] that [differentia]." No analogies replacing the definition.
- No history or context before the definition. Backstory, if any, belongs in the "how it works" section or is cut.
- The answer-first block and the TL;DR together function as the citation target. Treat them as the most important text in the article.
- Components table, "why it matters," and use cases sections each need to be self-contained chunks. A reader pulled in mid-page should understand the section on its own.
